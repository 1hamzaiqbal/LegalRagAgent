## 2. Method Breakdown

### snap_hyde (65.5%) — Previous Best

```
Step 1: SNAP — LLM answers question cold (1 call)
        "What is the answer to this torts question?" → reasoning + answer (B)

Step 2: HyDE — LLM generates a fake legal passage based on snap reasoning (1 call)
        "Write a legal reference passage that would verify this answer"
        → "Under the doctrine of res ipsa loquitur, a plaintiff must show..."

Step 3: RETRIEVE — Embed the fake passage, search corpus, cross-encoder reranks (0 calls)
        → 5 real passages from 686K corpus

Step 4: ANSWER — LLM answers fresh with retrieved passages (1 call)
        Input: passages + question (NO snap answer shown)
        → Final answer (C) — changed from snap!

Total: 3 LLM calls. Snap is used to steer retrieval but hidden from the final call.
```

### historical vectorless_direct (64.5%) — No Vector Store Needed

```
Step 1: SNAP — LLM answers question cold (1 call)

Step 2: GENERATE — LLM writes a legal reference note from memory (1 call)
        Structured as: governing rule, key exception, dispositive fact trigger,
        what would make a different answer plausible

Step 3: ANSWER — LLM answers fresh with generated note (1 call)
        Input: generated note + question (NO snap answer shown)

Total: 3 LLM calls. No embedding model, no ChromaDB, no cross-encoder.
```

### subagent_rag (66.0%) — NEW BEST

```
Step 1: SNAP — LLM answers question cold (1 call)
        → reasoning + answer (B). Used ONLY for gap analysis.

Step 2: GAP ANALYSIS — LLM identifies 1-2 evidence gaps (1 call)
        "What specific rules or facts am I uncertain about?"
        → "gap: liability transfer for continuing trespass | ask: does trespass liability follow the land?"

Step 3: PER-GAP SUBAGENT (2 calls per gap):
    3a. RAG retrieves 5 passages using the gap sub-question
    3b. SUBAGENT reads passages and writes a focused report:
        "The retrieved passages indicate that liability for a continuing trespass
         transfers to subsequent property owners. The original tortfeasor remains
         liable, but the new owner also has standing to seek removal..."

Step 4: MAIN AGENT answers with subagent reports (1 call)
        Input: reports + question
        NOT shown: snap answer, raw passages, answer letters
        → Final answer (C) — corrected!

Total: 5-7 LLM calls. Key: subagent filters noise, main agent never sees snap.
```

### Why subagent_rag beats snap_hyde

| Factor | snap_hyde | subagent_rag |
|---|---|---|
| Retrieval targeting | HyDE passage (one query) | Per-gap sub-questions (1-2 focused queries) |
| Evidence filtering | None — raw passages | Subagent reads & summarizes |
| Genre mismatch | Passages are doctrinal, question is narrative | Reports are in natural language |
| Snap anchoring | Hidden from final ✓ | Hidden from final ✓ |
| Noise | 5 passages, some irrelevant | Filtered into focused reports |

## 4. Three Failure Modes We Identified

**Noise:** Raw corpus passages contain irrelevant content. The 686K-passage corpus has legal doctrines mixed with case dicta. snap_rag_nosnap changes 35% of answers but with 0 net improvement (31 fixed = 31 broke) — the evidence changes the model's mind randomly.

**Anchoring:** When the model sees its own previous answer, it sticks with it regardless of evidence. gap_rag with snap: 2% changes. Without snap: significantly more. snap_hyde and vectorless_direct both hide the snap and get 19-27% changes.

**Genre mismatch:** Questions are fact-pattern narratives ("A person did X, Y, Z. What is the legal consequence?") while corpus passages are doctrinal rules ("The doctrine requires elements A, B, C..."). HyDE bridges this by generating passage-form queries. subagent_rag bridges it differently — the subagent summarizes passages into natural language reports.
